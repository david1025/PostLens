## font_awesome_flutter 10.x → 11.0.0 迁移

**问题：** Flutter 3.44.1 (Dart 3.12.1) 中 `IconData` 变为 `final class`，无法被继承。`font_awesome_flutter` 10.x 尝试继承 `IconData` 导致编译失败（多处 `error G8A8DE05F`）。

**解决：** 升级 `pubspec.yaml` 中 `font_awesome_flutter: ^10.7.0` → `^11.0.0`，然后修复 3 类兼容性问题：
1. 返回类型：6 个 `_getProtocolIcon` 函数 `IconData` → `FaIconData`
2. 参数类型：4 个文件中函数参数的 `IconData` → `FaIconData`
3. Widget 替换：5 个文件中 `Icon()` → `FaIcon()`

共修改 14 个文件，无新增 import 需求。

**第二轮修复（3个遗漏错误）：**
- `header_bar.dart:863`：`_HoverIconButton.icon` 字段 `IconData` → `FaIconData`，移除 `isFa` 标志，简化 FaIcon 逻辑

- `response_pane.dart:754`：`_buildStatusItem` 参数 `IconData? icon` → `FaIconData? icon`

## 16:19 - 修复 GET 请求 Query Parameters 输入时 URL 栏未实时更新

**问题**: 在 Params 标签页的草稿行（新建参数行）中输入 key/value 时，URL 栏只在失焦时才更新，不实时。

**根因**: `request_pane.dart` 中草稿行的 `onChanged` 只调用了 `_onDraftParamChanged()`（仅追加空行），没有触发 URL 更新逻辑。而已提交参数行的 `onChanged` 正确调用了 `updateParam()` → `_updateParamsAndSyncUrl()`。

**修复**:
- `request_provider.dart`: 新增 `setUrlOnly()` 方法和 `_skipParamParsing` 标志位，纯更新 URL 不解析参数（防止 URL 栏 onChanged 回环解析参数导致草稿行被覆盖）
- `request_pane.dart`: `_onDraftParamChanged()` 增加 `_syncUrlFromDraftParams()` 调用，实时计算包含草稿内容的 URL
- 新增 `_buildUrlFromDraftParams()` 辅助方法，合并已提交参数 + 草稿行内容构建完整 URL

## 17:13 - HTTP 请求逻辑验证 + 窗口关闭延迟分析

**HTTP 请求参数传递验证结果**：
- Query Parameters：UI表格 → request_provider.params → _interpolateRequest 变量替换 → _sendHttpRequest 过滤enabled → queryParameters Map → dio.request()。逻辑正确。
- Body 参数（form-data/x-www-form-urlencoded/raw）：每种类型独立 UI 控件和 Provider 字段，发送时 dio_client.dart 按 bodyType 分发。逻辑正确。
- GET/HEAD 方法正确跳过 body 处理。
- 注意：formData/urlEncodedData 的 Provider 方法未写入 `enabled` 字段，发送时用 `?? 'true'` 兜底。

**窗口关闭延迟根因**：
- onWindowClose() 三个阻塞操作：
  1. `await _persistTabsToDatabase()` — 500ms 防抖定时器活跃时阻塞 SQLite 写入
  2. `await stopCapture()` → proxy.stop() Rust IPC + disableSystemProxy() reg 系统命令(~100-500ms)
  3. `await windowManager.destroy()`
- 优化建议：_persistTabsToDatabase 和 stopCapture 改为 fire-and-forget（不 await），仅保留同步 _saveActiveRequestState。

## 17:24 - 修复窗口关闭逻辑

**修改文件**: `lib/presentation/pages/home_page.dart` `onWindowClose()`

**两处修改**：
1. **移除 `await _persistTabsToDatabase()`** — 数据由 500ms 防抖定时器实时保存，关闭时 cancel 定时器 + fire-and-forget 最终写入即可，不需要阻塞等待。
2. **抓包服务运行时弹出确认对话框** — 先检查 `captureState.isRunning`，若运行中则 `showDialog` 询问用户是否关闭。用户确认后 `await stopCapture()`（Rust IPC + reg 系统命令），点取消则不关闭窗口。
